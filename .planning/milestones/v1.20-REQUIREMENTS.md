## Requirements

### Proxy-Terminated mTLS Core
- **MTLS-01: Host configuration seam for mTLS header extraction**
  - **Description:** The host app must be able to configure how Lockspire extracts the proxy-forwarded client certificate (e.g., from an `X-Forwarded-Client-Cert` header) to accommodate various load balancers and infrastructure, without touching Cowboy internals.
- **MTLS-02: Pluggable certificate parsing and thumbprint calculation**
  - **Description:** Lockspire must extract the PEM-encoded certificate from the configured source, parse it, and compute the `x5t#S256` (SHA-256) thumbprint.

### Client Authentication
- **MTLS-03: `self_signed_tls_client_auth` Authentication**
  - **Description:** Support `self_signed_tls_client_auth` for client authentication at the token endpoint. A client authenticates by proving possession of the private key corresponding to a self-signed certificate, validated against the client's registered `jwks` or `jwks_uri`.
- **MTLS-04: `tls_client_auth` PKI Authentication**
  - **Description:** Support `tls_client_auth` for client authentication. A client authenticates via a PKI-issued certificate, validated against registered metadata attributes (e.g., `tls_client_auth_subject_dn`, `tls_client_auth_san_dns`, `tls_client_auth_san_uri`, `tls_client_auth_san_ip`, `tls_client_auth_san_email`).

### Certificate-Bound Access Tokens
- **MTLS-05: Token issuance with certificate binding**
  - **Description:** Access tokens (and refresh tokens, for binding) issued in an mTLS-authenticated session must include the `cnf` (confirmation) claim containing the `x5t#S256` thumbprint of the client's certificate.
- **MTLS-06: mTLS-bound Token Refresh**
  - **Description:** The token endpoint must enforce that the certificate presented during a refresh grant matches the `x5t#S256` thumbprint bound to the refresh token.

### Introspection & Discovery
- **MTLS-07: Introspection of mTLS-bound tokens**
  - **Description:** The introspection endpoint must expose the `cnf` claim for mTLS-bound tokens to allow resource servers to enforce the binding.
- **MTLS-08: Discovery Metadata Updates**
  - **Description:** Expose `tls_client_certificate_bound_access_tokens: true`, add `tls_client_auth` and `self_signed_tls_client_auth` to `token_endpoint_auth_methods_supported`, and expose `mtls_endpoint_aliases` in the `/.well-known/openid-configuration`.
