# v1.12 Token Exchange (RFC 8693) Requirements

## Token Exchange Foundation
- **TE-01 (Grant Type):** The `/token` endpoint must accept requests with `grant_type=urn:ietf:params:oauth:grant-type:token-exchange` and correctly parse `subject_token` and `actor_token`.
- **TE-02 (Token Validation & Downscoping):** The server must validate the incoming tokens (e.g. against the database) and enforce strict downscoping so that requested scopes do not exceed the original `subject_token` scopes.
- **TE-05 (Lineage & Storage):** The system must persist exchanged tokens in Ecto, tracking the lineage (e.g., via `grant_id` or equivalent) to ensure cascading revocation if the original token is revoked.

## Host App Integration
- **TE-03 (Policy Behaviour):** The system must expose a Behaviour (e.g., `Lockspire.TokenExchangeValidator`) that delegates the business logic for Impersonation and Delegation to the host app.

## Delegation Semantics
- **TE-04 (Delegation Claims):** When an `actor_token` is present, the issued access token must include a properly constructed `act` (actor) claim representing the delegation chain. The system must limit delegation depth (e.g., `max_delegation_depth: 1`) to prevent token bloat.
