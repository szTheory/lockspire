# Milestone v1.25: Support-Burden Reduction

**Status:** ✅ SHIPPED 2026-05-26

## Overview

This milestone reduced support cost on already-shipped advanced setup surfaces instead of adding protocol breadth. Lockspire now tells one coherent bounded-reactive remote-`jwks_uri` story across runtime behavior, doctor/admin/operator surfaces, docs, and release-contract proof, while tightening the canonical mTLS, protected-route, and logout guidance around the same support boundary.

## Milestone Summary

**Decimal Phases:**

- None

**Key Decisions:**

- Keep `docs/supported-surface.md` as the sole public authority for advanced-setup support claims and use helper-backed release-contract assertions only as drift fences.
- Treat remote `jwks_uri` rollover as bounded-reactive support only: cached reads, one forced refresh on mismatch, last-known-good preservation on refresh failure, and fail-closed current-request behavior.
- Keep runtime diagnosis separate from install verification by introducing `mix lockspire.doctor remote-jwks` instead of widening `mix lockspire.verify`.
- Keep the representative second advanced-setup proof surface narrow to the shipped Phoenix protected-route pipeline rather than broadening into generic protected-resource product scope.

**Issues Resolved:**

- Operators can now distinguish remote `jwks_uri` fetch, invalid-document, key-unavailable, and signature-invalid incidents without reading source code.
- Doctor, admin, docs, and runtime behavior now describe one identical remote-JWKS support story instead of parallel vocabularies.
- The mTLS, protected-route, and logout setup story is now explicit about what Lockspire owns versus what the host app or deployment environment owns.
- Release-contract and regression proof now fail loudly if the advanced-setup support story drifts from shipped behavior.

**Issues Deferred:**

- None recorded at milestone close. Future follow-on work should be triggered only by concrete adopter evidence.

**Technical Debt Incurred:**

- None recorded at milestone close.
