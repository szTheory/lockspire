# Requirements: Lockspire v1.6 — Device Authorization Grant (RFC 8628)

**Defined:** 2026-04-27
**Archived:** 2026-04-28
**Milestone:** v1.6 Device Authorization Grant (RFC 8628)
**Core Value:** A Phoenix team can become a trustworthy OAuth/OIDC provider inside its existing app without inventing the dangerous parts itself.

## Shipped Requirements

### CORE: Device Authorization Endpoint

- [x] **DEV-01**: Implement `POST /device/code` endpoint to initiate device authorization.
- [x] **DEV-02**: Generate high-entropy `device_code` and low-entropy `user_code` (Base20).
- [x] **DEV-03**: Create Ecto schema and storage for tracking pending device codes with strict TTLs (5-10 minutes).

### HOST: Verification UI Seam

- [x] **DEV-04**: Expose `GET /verify` and `POST /verify` integration seams for the host app to render consent and verification UI.
- [x] **DEV-05**: Prevent auto-submit on `verification_uri_complete` to mitigate remote phishing.
- [x] **DEV-06**: Provide documentation on rate-limiting the `/verify` endpoint for the host app (no built-in rate limiting).

### TOKEN: Polling & Issuance

- [x] **DEV-07**: Implement `POST /token` support for `grant_type=urn:ietf:params:oauth:grant-type:device_code`.
- [x] **DEV-08**: Handle `authorization_pending`, `slow_down`, and token issuance on the `/token` endpoint.
- [x] **DEV-09**: Enforce polling intervals and prevent database crush via efficient Ecto queries.
