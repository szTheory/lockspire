# Milestone v1.16: Embedded Adoption Hardening & Sigra Golden Path

**Status:** ✅ SHIPPED 2026-05-07
**Phases:** 63-66
**Total Plans:** 13

## Overview

v1.16 shifts Lockspire from adding more protocol breadth to making its intended embedded Phoenix adoption path boring, trustworthy, and repo-proven. The milestone hardens the canonical install and upgrade flow, adds first-party host diagnostics, proves the Sigra-backed generated-host path end to end, reconciles package and support truth around the 1.0.0 posture, and retires the remaining conformance overclaim debt by narrowing current proof to repo-native evidence.

## Phases

**Depends on**: None (v1.15 completed)
**Requirements**: HOST-01, HOST-02, HOST-03
**Plans**: 4 plans

## Milestone Summary

**Decimal Phases:**

- None

**Key Decisions:**

- Keep `mix lockspire.install` as the one canonical embedded onboarding path and limit `--sigra-host` to host-guidance comments rather than a second topology.
- Separate Lockspire-managed scaffolding from host-owned seams explicitly and use a manifest plus checksum checks to keep upgrades safe.
- Treat `docs/supported-surface.md` as the sole public support contract, with README, SECURITY, and maintainer docs subordinate to it.
- Retire historical external-suite conformance output as non-claim audit context and keep current public proof repo-native.

**Issues Resolved:**

- Removed ambiguity about which generated files Lockspire owns versus which seams the host app must implement.
- Added a canonical diagnostics and upgrade path for embedded hosts instead of relying on install-time folklore.
- Replaced the stale release-truth split between docs and package metadata with one coherent 1.0.0 posture.
- Closed the public conformance overclaim by demoting external-suite artifacts below current repo-owned proof.

**Issues Deferred:**

- `SEED-001`: cut the next real public Lockspire release now that the embedded host path and release truth are aligned.
- Historical `37-VERIFICATION.md` remains preserved as non-authoritative audit context, not as a current blocker or support claim.

**Technical Debt Incurred:**

- No new material product-surface debt was accepted at close; the remaining deferred work is milestone-selection and release execution, not support-contract ambiguity.
