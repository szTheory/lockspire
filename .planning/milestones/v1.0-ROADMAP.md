# Milestone v1.0: Embedded OAuth/OIDC Provider Foundation

**Status:** ✅ SHIPPED 2026-04-23
**Phases:** 1-6
**Total Plans:** 25

## Overview

Lockspire completed its first milestone as a focused embedded OAuth/OIDC provider library for Phoenix. The milestone delivered the install path, authorization code + PKCE core, OIDC discovery and lifecycle endpoints, operator/admin workflows, and the security/observability posture needed for a credible embedded library.

This archive records the milestone as complete from a planning and implementation standpoint. Public release posture remains **v0.1 preview**, not a full `1.0` release claim, because the repo still needs repeated green release gates, repo-wide Credo cleanup, and trusted Hex publish verification.

## Milestone Summary

**Key Accomplishments:**

- Delivered an embedded Phoenix install path that preserves host ownership of accounts, branding, layouts, login UX, and policy.
- Shipped authorization code + PKCE with exact-match redirect validation, durable consent/interaction state, and secure token exchange defaults.
- Completed the OIDC provider surface with discovery, JWKS, userinfo, refresh rotation, revocation, and introspection.
- Built calm LiveView-native operator workflows for clients, consents, tokens, and signing keys.
- Added durable telemetry and audit infrastructure, centralized redaction, and protocol/delivery-edge negative-path coverage.
- Established docs, CI, release automation, security reporting, and supported-surface guidance for a public preview posture.

**Key Decisions:**

- Lockspire ships as a separate companion library, not a Sigra module.
- The product remains an embedded library, not a required standalone auth service.
- Host seams stay narrow and explicit: account resolution, claims, login redirects, branding, and product policy remain host-owned.
- Protocol core, storage, generators, Plug/Phoenix integration, and operator UI remain separate internal boundaries.
- Public claims stay at a serious preview bar until repeated green release gates and trusted release verification exist.
- Sigra ecosystem guidance remains a parallel track and does not block the core Lockspire milestone.

**Issues Resolved:**

- Closed the embedded-library install and host-seam boundary without inventing a hosted auth service.
- Closed the end-to-end authorization code + PKCE and consent lifecycle.
- Closed OIDC discovery/JWKS/userinfo and durable refresh/revocation/introspection lifecycle support.
- Closed admin/operator workflows and the security redaction/audit baseline needed for supportability.

**Issues Deferred:**

- Repo-wide Credo debt still blocks a clean green `mix ci` lane.
- Trusted Hex authentication is still required to exercise publish dry-runs and the final release path.
- Repeated green release-gate evidence is still needed before making a public `1.0` claim.

**Technical Debt Incurred:**

- Planning docs drifted behind execution during late-phase closeout and required reconciliation at archive time.
- Release-readiness proof is explicit but still partially environment-dependent because Hex publishing cannot be exercised from an untrusted local shell.
