<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| host claim material -> protocol JWT | Untrusted host-provided claim maps cross into Lockspire-owned ID token assembly |
| protocol timestamp input -> signed token | Invalid timestamp types could become publicly visible token claims |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-37-01 | Tampering | `Lockspire.Host.Claims` | mitigate | Reserve `auth_time` in `@protocol_claims` so host maps cannot overwrite protocol-owned freshness truth. |
| T-37-02 | Information Disclosure | `Lockspire.Protocol.IdToken` | mitigate | Reject invalid auth_time input before signing and emit only integer Unix timestamps from `%DateTime{}` values. |
| T-37-03 | Repudiation | `Lockspire.Protocol.IdToken` | mitigate | Add direct regression tests that decode the JWT payload so the claim shape is provable and stable. |
</threat_model>
