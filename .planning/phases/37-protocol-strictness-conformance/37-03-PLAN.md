<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| host session context -> `AuthorizationFlow` | Host-owned session/account state is read by Lockspire for protocol decisions |
| durable interaction state -> token exchange | Persisted OIDC request metadata flows into ID token issuance |
| authorization outcome -> browser redirect | Silent-auth failures are exposed to client callbacks |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-37-07 | Spoofing | silent session inspection | mitigate | Require explicit `subject_context[:auth_time]` input for fresh-auth truth and never infer it from generic session reuse. |
| T-37-08 | Tampering | `Interaction` persistence | mitigate | Persist `auth_time`, `max_age`, and `auth_time_requested` durably in the interaction row and use that record as the token-issuance source of truth. |
| T-37-09 | Repudiation | silent authorization errors | mitigate | Return deterministic `login_required`, `consent_required`, and `interaction_required` tuples from protocol code and cover them with unit/controller tests. |
| T-37-10 | Elevation of Privilege | `prompt=none` branch | mitigate | Short-circuit silent failures before any login or consent redirect path can execute. |
</threat_model>
