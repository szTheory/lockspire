<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser query params -> `AuthorizationRequest.validate/1` | Untrusted user input crosses into protocol validation |
| validated request -> Phoenix controller redirect | Redirect-safe errors cross back into the user agent and client callback |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-37-04 | Spoofing | `redirect_uri` validation | mitigate | Preserve exact string membership checks and keep invalid redirect URIs on the browser-safe error surface. |
| T-37-05 | Tampering | `prompt` / `max_age` / `claims` parsing | mitigate | Parse `prompt=none`, `max_age`, and `claims` centrally in `AuthorizationRequest` with explicit reason codes and no permissive fallback. |
| T-37-06 | Information Disclosure | controller redirect handling | mitigate | Only redirect with OAuth error params after a trusted exact `redirect_uri` match has already succeeded. |
</threat_model>
