## Gaps Summary

Two blockers prevent Phase 37 goal achievement:

**Blocker 1 — CONF-04 not satisfied (OIDF suite never ran).** The Plan 04 conformance harness was built and wired correctly — the `mix conformance.phase37` alias, the shell script, the `phase37-plan.json`, and the GitHub Actions workflow are all substantive and properly connected. However, the actual external OIDF Docker suite was never executed. The harness ran with `LOCKSPIRE_PHASE37_SKIP_SUITE=true` because Docker was unavailable in the worktree environment. The artifacts in `.artifacts/conformance/phase37/` are stub outputs: `run-summary.json` explicitly states the suite was skipped, and `exported_files` is empty. Both REQUIREMENTS.md (CONF-04 checkbox `[ ]`) and ROADMAP.md (37-04-PLAN.md `[ ]`) confirm this requirement is unmet. This is the definition of SC4 not achieved: "An automated test pipeline *successfully runs* against the OIDF Conformance Test Suite without failures."

**Blocker 2 — token_exchange_test.exs:131 fails due to test DB pollution.** The conformance harness (`run_phase37_suite.sh:128-143`) inserted a signing key with `kid="phase37-conformance-kid"` and `status=:active` into the test database outside Ecto sandbox isolation. The existing test at line 131 ("issues an RS256 id token for openid code flow using the linked interaction nonce") queries `SELECT * FROM lockspire_signing_keys WHERE status='active' ORDER BY inserted_at LIMIT 1` and gets the polluted key instead of the test-published `kid-openid`. This is a Phase 37 regression: the test was passing before Plan 04 execution. The Plan 03 must-have "Phase 37's token-exchange changes do not regress existing nonce pass-through" is therefore falsified — the failure is a different regression, but it represents an identical broken state.

**Resolution path:** (1) Run `MIX_ENV=test mix conformance.phase37` without `LOCKSPIRE_PHASE37_SKIP_SUITE=true` in a Docker-capable environment and commit real proof artifacts. (2) Clean the `phase37-conformance-kid` key from the test database (or ensure the harness uses a separate DB / wraps its key insertion in a transaction that does not persist), then confirm `mix test test/lockspire/protocol/token_exchange_test.exs` exits 0.
