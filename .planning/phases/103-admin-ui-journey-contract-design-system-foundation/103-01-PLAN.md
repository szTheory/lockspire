# Phase 103-01 Plan: Admin Journey Contract + Component Foundation

## Objective

Create the reusable admin component and CSS foundation needed for the v1.28 UI polish milestone, then migrate enough existing screens to prove the foundation is usable without changing protocol behavior.

## Tasks

1. Expand `AdminComponents` with shared primitives:
   - `admin_button`
   - `action_bar`
   - `alert`
   - `description_list`
   - `summary_stat`
   - `resource_list` / `resource_item`
   - `badge_group`
   - `confirmation_panel`

2. Extend `Admin.CSS`:
   - Canonical button modifiers and disabled/busy states.
   - Action bar, detail section, badge group, resource row parts, confirmation panel.
   - `prefers-reduced-motion` guard.
   - Broader focus-visible coverage.
   - Responsive rules for action bars, resource rows, and long identifiers.

3. Migrate foundation targets:
   - Clients index filter/rows.
   - IAT list/new.
   - Existing buttons/actions that currently rely on generic/raw button styling in those targets.

4. Strengthen contracts:
   - Design-system contract test requires new CSS primitives.
   - Test no generic `class="button"` patterns in admin LiveViews.
   - Test migrated screens render journey copy and expected classes.

5. Verify:
   - Format touched files.
   - `MIX_ENV=test mix compile --warnings-as-errors`
   - Focused admin tests.

## Acceptance Criteria

- Shared component API exists and is used by migrated screens.
- Migrated screens contain no inline layout/color styling except approved local exceptions.
- Buttons, action bars, resource rows, confirmation panels, and description lists have reusable CSS classes.
- Mobile-friendly behavior is covered by CSS and browser spot checks.
- No protocol/storage behavior changes.
