<objective>
Correct the canonical public support contract so it matches the shipped DCR/logout metadata behavior without widening Lockspire's logout product claim.
</objective>

<scope>

<threat_model>
## Threats

- `T-87-01` The canonical support page may keep a stale unsupported claim, causing public support truth to contradict shipped behavior.
- `T-87-02` A support-truth fix could accidentally broaden the milestone into a new logout-runtime claim or overstate front-channel reliability.

## Mitigations

- Update the supported DCR/RFC 7592 bullet so it explicitly covers management of the four existing logout propagation metadata fields per D-01 and D-06.
- Replace the stale out-of-scope bullet with wording that keeps the boundary narrow: DCR manages existing logout propagation metadata only; it does not add a new logout runtime.
- Keep one canonical asymmetry statement on this page per D-07 and D-08: back-channel logout is durable, front-channel logout is best effort only.
- Preserve the separation between post-logout redirect URIs and logout propagation metadata per D-09.
</threat_model>
