<objective>
Implement the `Lockspire.Plug.VerifyToken` plug.

Purpose: This "soft validation" plug processes the incoming `Authorization: Bearer` header, fetches the appropriate public key from `KeyCache`, verifies the JWT signature and time constraints, and securely sets an `%AccessToken{}` struct into `conn.assigns`. It explicitly avoids halting the connection, following idiomatic "Two-Plug" Elixir auth patterns.

  <name>Task 1: Implement VerifyToken Plug</name>
  <files>lib/lockspire/plug/verify_token.ex, test/lockspire/plug/verify_token_test.exs</files>
  <behavior>
    - If `Authorization` header is missing or not a Bearer token: assign `%AccessToken{error: :missing_token}`.
    - If Bearer token is malformed JWT: assign `%AccessToken{error: :invalid_token}`.
    - If JWT `kid` header is missing or `KeyCache.get_key(kid)` fails: assign `%AccessToken{error: :invalid_token}`.
    - If `JOSE.JWT.verify_strict/3` fails against an allowed algorithm list (`["RS256", "ES256", "PS256"]`): assign `%AccessToken{error: :invalid_token}`.
    - If `exp` is in the past or `nbf` is in the future: assign `%AccessToken{error: :invalid_token}`.
    - If valid: assign `%AccessToken{token: token, claims: claims, client_id: claims["client_id"]}` to `conn.assigns[:access_token]`.
    - Never calls `halt(conn)`.
  </behavior>
  <action>
    Implement `Lockspire.Plug.VerifyToken` implementing the `Plug` behaviour.
    Use `Plug.Conn.get_req_header/2` to extract the token.
    Use `JOSE.JWT.peek_protected/1` and `JOSE.JWS.to_map/1` safely to extract the `kid` without crashing on bad input.
    Delegate to `Lockspire.KeyCache.get_key/1`.
    Validate the signature. Verify standard JWT time claims (`exp` and `nbf`) explicitly, as JOSE does not enforce time automatically.
    Reference the analog files `lib/lockspire/mtls/plug.ex` and `lib/lockspire/protocol/fapi20_enforcer_plug.ex` from PATTERNS.md.
  </action>
