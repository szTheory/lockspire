<objective>
Build the core data structure (`AccessToken`) and high-speed ETS cache for signing keys (`KeyCache`) to enable microsecond-latency JWT validation without hitting the database on every request.

Purpose: Resource servers must validate tokens at high volume. Fetching active keys from the database per-request will destroy connection pool throughput. An ETS cache initialized and maintained by a GenServer provides the required concurrent read performance.
Output: `AccessToken` struct, `KeyCache` GenServer, and updated application supervisor.
</objective>
