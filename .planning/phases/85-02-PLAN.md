<objective>
Carry accepted logout propagation metadata from RFC 7591 input into the persisted self-registered client record, preserving the existing durable client/logout truth model.
</objective>

<scope>
This plan covers only the registration create path. It does not change RFC 7592 full-replace update semantics or operator workflows.
</scope>

<threat_model>
## Threats

- `T-85-03` DCR create may validate logout metadata but fail to persist it, creating a false success contract for self-service clients.
- `T-85-04` Persistence wiring could bypass typed client fields or leak unnormalized input into durable state.

## Mitigations

- Build the persisted `%Lockspire.Domain.Client{}` from normalized logout metadata, not raw wire values.
- Reuse the existing typed client fields and Ecto schema columns already consumed by logout propagation.
- Add round-trip proof at both protocol and repository layers.
</threat_model>

  - Verify: successful registration returns a client struct carrying the normalized values.
