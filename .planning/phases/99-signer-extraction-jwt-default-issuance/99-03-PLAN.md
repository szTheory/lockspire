<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| issuance path → signer | The signer receives a built %Token{}; it must not let any client-controlled value override the signing alg or restricted claims. |
| signer → JWKS/private key store | Decrypted private JWK is held in-memory only; it must never reach logs. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-99-06 | Spoofing / Tampering | `alg=none` / alg confusion in the minted JWT | mitigate | Header `alg`+`kid` taken ONLY from `fetch_active_signing_key/1`; no client-controlled alg; JOSE pins the header. Never emit `none` (AGENTS.md). |
| T-99-07 | Information Disclosure | Key material / decoded JWK / raw token in logs on the error path | mitigate | The 500 error path logs `inspect(reason)` only — no `private_jwk`, no decoded JWK, no raw token. Test asserts captured logs contain no key material. |
| T-99-08 | Elevation of Privilege | Audience confusion — token minted for client A accepted elsewhere | mitigate | `aud` derives strictly from `%Token{}.audience` (set by per-path `validate_requested_resources` in Plans 04/05) or `[client_id]`; the signer cannot be coerced into an attacker-chosen `aud` because it never reads `resource` from raw request params. The restricted-claim drop `~w(iss sub aud exp iat jti client_id)` prevents custom-claim override of `aud`. |
| T-99-09 | Spoofing | `typ` confusion (JWT vs at+jwt) | mitigate | Signer sets exact `typ: "at+jwt"`; Phase 98 verifier rejects other `typ`. Round-trip asserted in tests. |
| T-99-10 | Spoofing (sender-constraint downgrade) | `cnf` dropped from minted JWT | mitigate | Signer copies `%Token{}.cnf` when present so Phase 100 can verify DPoP/mTLS binding; dropping it would silently unbind. |
| T-99-SC | Tampering | npm/pip/cargo installs | accept | No packages added in Phase 99. |
</threat_model>
