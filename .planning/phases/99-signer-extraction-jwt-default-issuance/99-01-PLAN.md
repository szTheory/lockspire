<objective>
Lay the storage + domain foundation for the access-token format policy: a runtime-editable server-wide `access_token_format` (default `:jwt`) on `ServerPolicy` and a nullable per-client `access_token_format` override on `Client`, plus the migration that adds both `:text` columns and the `Admin.ServerPolicy.put_access_token_format/1` runtime setter.

Purpose: FORMAT-01 requires operators to flip the deployment-wide token shape at runtime, and FORMAT-02 requires a per-client override that wins independently of the server default. Both the signer (Plan 03) and the admin UI (Plan 06) read these fields, so this foundation must land first.

Output: One migration, two record schemas + two domain structs extended, and one new admin context function — no signing or UI behavior yet.
</objective>

| Boundary | Description |
|----------|-------------|
| operator (admin/console) → ServerPolicy | An operator setting the deployment-wide token format is trusted but their input string must be validated to a known enum. |
| operator (admin form) → Client record | Per-client override values cross from the admin form into the persisted client record via `update_changeset/2`. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-99-01 | Tampering | `put_access_token_format/1` accepting an arbitrary value | mitigate | Normalize via `normalize_access_token_format/1` to `:jwt`/`:opaque` only; unknown → `{:error, _}`. Ecto.Enum rejects out-of-set values at changeset cast. |
| T-99-02 | Elevation of Privilege | Default-flip blast radius (server default now `:jwt`) | accept | Existing opaque tokens already in `lockspire_tokens` remain valid and continue to back `/userinfo`+`/introspect` (untouched). The migration only adds columns; it does not invalidate stored tokens. Per-client `:opaque` opt-in remains available (FORMAT-02). |
| T-99-03 | Tampering | Ecto.Enum field unpaired with a `:text` DB column (Pitfall 6) | mitigate | Migration adds `:text` columns in Task 1 before the Ecto.Enum fields are declared; `:jwt` value persists/loads as `"jwt"` correctly. |
| T-99-SC | Tampering | npm/pip/cargo installs | accept | No packages added in Phase 99 (RESEARCH "Package Legitimacy Audit: Not applicable"). No install task exists. |
