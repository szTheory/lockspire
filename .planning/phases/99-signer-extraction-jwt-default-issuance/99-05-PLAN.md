## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| presented refresh token → rotated token | The subject (`account_id`) must be carried from the presented token; a `nil` subject would mint an unverifiable token. |
| RFC 8693 subject token → minted exchange token | The exchange path's `aud`/custom-claim assembly must not let custom claims override restricted claims. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-99-14 | Repudiation / Tampering | Refresh-issued JWT with `sub: nil` (rotated `account_id: nil`, Pitfall 5) | mitigate | Source `sub` from `presented_refresh_token.account_id` before signing; test asserts non-nil `sub` and round-trip against the Phase 98 `:missing_sub` check. |
| T-99-15 | Elevation of Privilege | AUD-03 regression — exchange `aud` becomes a list, breaking downstream resource-servers that match a string | mitigate | The signer's exchange-path mode keeps a bare-string `aud == client_id`; `rfc8693_exchange_test.exs:192` is the regression sentinel and must stay green. |
| T-99-16 | Tampering | Duplicated signing logic left behind → divergent claim/typ assembly, alg drift | mitigate | The `JOSE.*` block and key-fetch helpers are deleted from `rfc8693_exchange.ex` (grep gate returns 0); all signing flows through the single `AccessTokenSigner` site (SC5). |
| T-99-17 | Tampering | Custom claims overriding `iss`/`sub`/`aud`/`exp`/`iat`/`jti`/`client_id` on the exchange path | mitigate | The restricted-claim drop `~w(iss sub aud exp iat jti client_id)` is preserved in the signer's exchange mode; custom claims cannot override protocol claims. |
| T-99-18 | Information Disclosure | Key material in logs on the exchange error path | mitigate | The preserved error path logs `inspect(reason)` only; no key material (AGENTS.md redaction). |
| T-99-SC | Tampering | npm/pip/cargo installs | accept | No packages added in Phase 99. |
