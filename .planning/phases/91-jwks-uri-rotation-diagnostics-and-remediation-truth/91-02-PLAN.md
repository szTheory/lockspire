<threat_model>
## Threats

- `T-91-05` A new support command could blur into install verification and confuse operators about when to use `mix lockspire.verify` versus remote-JWKS diagnosis.
- `T-91-06` CLI and admin surfaces could each summarize incidents differently, recreating the fragmented support story the phase is trying to remove.
- `T-91-07` Operator output could become too low-level, secret-adjacent, or speculative about upstream intent.
- `T-91-08` Admin could become the de facto authority for remote-JWKS truth instead of a consumer of the shared runtime diagnostics model.

## Mitigations

- Introduce a dedicated doctor-style remote-JWKS task with help text that explicitly distinguishes runtime incident diagnosis from install verification.
- Render CLI and admin summaries from the same shared diagnostic shape and stable incident classes.
- Keep the output short and remediation-oriented: incident class, key facts, ownership split, and next step.
- Add tests that pin the wording boundary between the doctor task, admin surface, and the underlying diagnostics model.
</threat_model>
