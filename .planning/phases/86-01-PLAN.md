<objective>
Make RFC 7592 management update persist the four logout propagation fields under true full-replace semantics so self-service clients can set, replace, and clear them through the shipped DCR seam.
</objective>

<scope>
This plan is limited to the management update write path. It does not broaden the logout runtime itself or change support-surface wording.
</scope>

<threat_model>
## Threats

- `T-86-01` RFC 7592 PUT may validate logout metadata successfully but fail to persist it, creating a false management contract.
- `T-86-02` Partial field application could leave stale logout metadata behind and violate full-replace semantics.

## Mitigations

- Normalize the logout metadata once per update request and assign all four typed fields explicitly.
- Treat omitted logout fields as replacement clears, not "leave existing state untouched".
- Prove set, replace, and clear behavior with focused management protocol tests.
</threat_model>

<tasks>
- [ ] `86-01-01` Thread normalized logout propagation fields into `RegistrationManagement.apply_metadata_to_client/2`.
  - Files: `lib/lockspire/protocol/registration_management.ex`, `lib/lockspire/admin/clients.ex`
  - Action: reuse `Admin.Clients.normalize_logout_metadata/1` so update persistence writes `backchannel_logout_uri`, `backchannel_logout_session_required`, `frontchannel_logout_uri`, and `frontchannel_logout_session_required` onto the rebuilt `%Client{}`.
  - Verify: successful update returns a client carrying the normalized logout metadata.

- [ ] `86-01-02` Enforce full-replace clearing semantics for optional logout fields.
  - Files: `lib/lockspire/protocol/registration_management.ex`, `test/lockspire/protocol/registration_management_test.exs`
  - Action: ensure omitted logout URIs clear prior persisted values and the paired `*_session_required` booleans reset to `false` when the URI is absent.
  - Verify: a client that starts with logout metadata can be updated with plain `valid_metadata()` and ends with both URIs `nil` and both booleans `false`.

- [ ] `86-01-03` Add focused management-update coverage for set, replace, and clear flows.
  - Files: `test/lockspire/protocol/registration_management_test.exs`, `test/support/fixtures/dcr_fixtures.ex`
  - Action: add fixtures and protocol tests for initial set, replacement with different valid values, and full clear by omission.
  - Verify: `mix test test/lockspire/protocol/registration_management_test.exs`.
</tasks>

<verification>
- Run `mix test test/lockspire/protocol/registration_management_test.exs`.
- Confirm successful RFC 7592 updates can set, replace, and clear the four logout propagation fields.
- Confirm clearing behavior follows full-replace semantics instead of preserving stale stored values.
</verification>

<success_criteria>
- RFC 7592 management update persists valid logout propagation metadata onto the self-registered client.
- Omitting optional logout metadata from a management PUT clears previously stored values.
- The write path remains on the typed client fields that already drive runtime and operator truth.
</success_criteria>
