<threat_model>
## Threats

- `T-89-04` Discovery could advertise `client_secret_jwt` on endpoints that do not actually use the shared direct-client verifier.
- `T-89-05` One issuer-wide JWT signing-alg list could falsely imply that `HS256` applies to `private_key_jwt` or that asymmetric algorithms apply to `client_secret_jwt`.
- `T-89-06` FAPI-effective issuer posture could continue publishing `client_secret_jwt` or `HS256`, contradicting the shipped runtime denial.

## Mitigations

- Keep discovery publication endpoint-local and mounted-route-based, reusing the existing token/revocation/introspection truth shape.
- Build signing-alg publication from the published methods on each endpoint: add `HS256` only when `client_secret_jwt` is advertised, add asymmetric algorithms only when `private_key_jwt` is advertised.
- Gate the symmetric JWT publication path on the effective issuer security profile so FAPI profiles suppress both the method and its `HS256` algorithm.
</threat_model>
