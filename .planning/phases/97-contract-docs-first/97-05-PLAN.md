Helper signatures and contracts:

- `extract_canonical_pipeline!(path :: String.t(), kind :: atom()) :: binary` — reads the file at `path`, runs `Regex.run/3` with the marker-region capture, raises with a clear message naming `path` if extraction fails (returns `nil` or matches an empty region), otherwise passes the captured bytes through `normalize/2` and returns the result.

- `normalize(bytes :: binary, kind :: atom) :: binary` — kind-aware normalization. Two clauses:
  - `normalize(bytes, kind) when kind in [:python_commented, :elixir_in_commented_heredoc]` — applies `String.replace("\r\n", "\n")` then splits on `"\n"`, applies `String.replace_prefix(&1, "# ", "")` per line, joins on `"\n"`, applies strip-uniform-indent (defined below), applies `String.replace(~r/[ \t]+$/m, "")`.
  - `normalize(bytes, _kind)` (catch-all for `:elixir_in_markdown_fence` and `:elixir`) — applies `String.replace("\r\n", "\n")`, applies strip-uniform-indent, applies `String.replace(~r/[ \t]+$/m, "")`. NO `# ` prefix strip.

- `strip_uniform_indent(bytes :: binary) :: binary` — private helper of `normalize/2`. Splits on `"\n"`; computes minimum leading-whitespace count across non-blank lines (`Enum.min` over `String.length(leading)` where leading is `Regex.run(~r/^[ \t]*/, line) |> List.first()`); strips that many characters from the start of every line; joins on `"\n"`. If all lines are blank (defensive), returns the input unchanged.

- `canonical_hash!(path :: String.t(), kind :: atom) :: binary` — calls `extract_canonical_pipeline!(path, kind)`; raises with a clear message if the result does not contain the substring `Lockspire.Plug.VerifyToken` (RESEARCH Pitfall 4 sanity guard); raises with a clear message if `path` ends with `.ex` AND the result contains `<%` or `<%=` (RESEARCH Pitfall 3 EEx-tag guard); returns `:crypto.hash(:sha256, bytes)`.

Regex pattern for marker extraction (verbatim — `[ms]` flags MUST both be present per RESEARCH "Don't Hand-Roll" advice and existing-precedent imitation):

```
~r/# BEGIN LOCKSPIRE_PROTECTED_PIPELINE\n(.*?)\n[ \t]*# END LOCKSPIRE_PROTECTED_PIPELINE/ms
```

The `[ \t]*` before `# END` absorbs any per-file leading-whitespace indent on the END marker line (the BEGIN marker line's leading whitespace is NOT part of the capture because the regex starts AT the BEGIN literal). The capture-group bytes start AFTER the `\n` following BEGIN and end BEFORE the `\n` preceding END — so the capture is exactly the canonical region interior, no markers, no trailing or leading newline.
