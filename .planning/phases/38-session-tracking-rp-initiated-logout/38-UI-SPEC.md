# Phase 38 — UI Design Contract

> Visual and interaction contract for Phase 38: Session Tracking & RP-Initiated Logout.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Scope of UI Work

Three discrete UI surfaces are introduced or extended in this phase:

1. **"You have been signed out" page** — plain controller render (`EndSessionHTML.logged_out`). Minimal static page shown when no `post_logout_redirect_uri` is registered or provided.
2. **Token detail view — `sid` field** — view-only `<p>` row added to the existing `tokens_live/show.ex` section card, matching the pattern of `jti`, `family_id`, and `cnf` rows.
3. **Client show + edit form — `post_logout_redirect_uris` field** — a new `:logout_uris` mode section card and a textarea input following the exact pattern of the existing `:redirects` mode for `redirect_uris`.

No new LiveView modules, no new pages beyond the logged-out page, no new nav items, and no session browsing UI (handled in Phase 39).

## Logged-Out Page Contract

File: `lib/lockspire/web/controllers/end_session_html/logged_out.html.heex`

**Layout:** Plain controller render — NOT wrapped in `AdminLayoutLive.shell` (this page is user-facing, not admin-facing). It is the only Lockspire-owned page in Phase 38 that is not an admin surface.

**Structure:**
```
<section class="lockspire-logged-out">
  <h1>You have been signed out</h1>
  <p>Your session has ended. You may close this tab or return to the application.</p>
</section>
```

**Rationale:** D-24 specifies "minimal static template." The page must be host-overridable via Phoenix template override. It must not use the admin nav shell — the user who sees this page may be an end user, not an operator.

**Classes:** One new class `lockspire-logged-out` — scoped outside `lockspire-admin-*` namespace intentionally. Minimal styling: centered container, `max-width: 480px`, `margin: 4rem auto`, `padding: 1.5rem` (24px), `text-align: left`.

## Interaction States

### Token Detail View (`tokens_live/show.ex`)

| State | Behavior |
|-------|---------|
| `sid` present | Renders `<code>{sid_value}</code>` |
| `sid` nil (pre-Phase-38 token) | Renders `<code>Not recorded</code>` |
| Token not found | Existing `AdminComponents.empty_state` with "Token not found" + "Lockspire could not load that lifecycle token from durable storage." (unchanged) |

### Client Show View (`clients_live/show.ex`)

| State | Behavior |
|-------|---------|
| `post_logout_redirect_uris` is empty list | Renders empty `<ul>` with heading present (no empty state message — matching `redirect_uris` pattern) |
| `post_logout_redirect_uris` has entries | Renders `<li>` per URI |
| `:logout_uris` edit mode active | Shows new section card with textarea form |
| Save succeeds | Section card dismisses; updated list renders in show view |
| Save fails (validation error) | Existing `lockspire-admin-errors` list renders inside the form section card |

### Logged-Out Page (`end_session_html/logged_out.html.heex`)

| State | Behavior |
|-------|---------|
| Rendered (the only state) | Static page with heading + body copy. No interactive elements. No form. No LiveView. |

## Accessibility

- The `logged_out.html.heex` page must have a document `<title>` of "Signed out — Lockspire" set via `conn |> put_resp_content_type(...)` or the controller's `put_view`.
- The `sid` field row uses `<p>Session ID: <code>...</code></p>` — label and value are inline, matching all other token detail rows. No additional ARIA attributes needed.
- The new textarea for `post_logout_redirect_uris` must have a `<label for="...">` matching `id="client_post_logout_redirect_uris"` — matching the `redirect_uris` textarea pattern exactly.
- The new `:logout_uris` action link must have descriptive text "Edit post-logout redirect URIs" (not "Edit" alone) so it is distinguishable from adjacent "Edit redirect URIs".
