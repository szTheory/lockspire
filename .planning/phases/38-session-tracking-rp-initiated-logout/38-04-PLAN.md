## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| RP/client → discovery endpoint | Discovery is read-only; no input processing — low risk surface |
| operator → admin UI (clients form) | post_logout_redirect_uris input from admin form goes through Ecto changeset validation and split_lines |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-38-12 | Tampering | post_logout_redirect_uris admin form | mitigate | `update_changeset/2` casts `{:array, :string}` — Ecto rejects non-list values. `split_lines/1` in `redirect_attrs/2` normalizes textarea newline format before changeset. Exact-match enforcement at logout-time (EndSessionProtocol) is the runtime gate. |
| T-38-13 | Information Disclosure | discovery BCL/FCL false flags | accept | Publishing `false` flags is truthful and required by OIDC spec (D-19). No sensitive data exposed. |
| T-38-14 | Information Disclosure | sid visible in admin token detail | accept | Admin UI is operator-only (authenticated). sid is an opaque random identifier, not a secret per se. |
| T-38-15 | Spoofing | generator stub /logout path | accept | Generator template is a stub with a TODO comment. The host is responsible for implementing the real session-clearing route. Risk is developer error, not protocol attack. |
</threat_model>
