<objective>
Persist the newly issued exchanged tokens into the database, preserving token lineage for cascading revocation.

Purpose: Lifecycle management. Token exchange creates derived tokens that must share the same `family_id` as the original token so that revoking the parent or family automatically revokes the derived tokens.
Output: Token exchange requests succeed, returning a properly formatted response and a securely persisted token with lineage intact.
</objective>
