| Boundary | Description |
|----------|-------------|
| Host browser seam -> verification protocol | Untrusted user-entered `user_code` crosses into typed lookup semantics. |
| Verification protocol -> repository | Approval and denial cross into durable state transitions that must resist concurrent mutation. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-31-03 | Spoofing | `lib/lockspire/protocol/device_verification.ex` | mitigate | Require lookup-by-code followed by opaque-handle mutation so host code cannot approve directly from a prefilling GET. |
| T-31-04 | Tampering | `lib/lockspire/protocol/device_verification.ex` | mitigate | Delegate final approve/deny correctness to expected-state repository transitions and preserve typed stale outcomes on retries. |
| T-31-05 | Information Disclosure | `lib/lockspire/protocol/device_authorization.ex` | mitigate | Emit `verification_uri_complete` as prefill-only output and leave neutral invalid-or-expired collapse to the host seam instead of leaking existence semantics here. |
</threat_model>
