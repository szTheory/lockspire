<objective>
Verify, complete, and test the existing uncommitted Admin LiveView surfaces for managing
`security_profile` — both the global ServerPolicy page (`/admin/policies/security-profile`)
and the per-client override (`/admin/clients/:client_id/security-profile`). Add the
visible "mixed-mode override" warning that T-41-03/T-41-07 require.

Purpose: FAPI-01 ("Provide a single `security_profile: :fapi_2_0_security` option to
enable strict mode globally or per-client") is operator-facing. Without an Admin UI, the
toggle is unreachable from the host application. The uncommitted scaffolding already
implements ~90% of this; this plan ratifies it with tests, ensures the policy_nav tab is
wired into all four policy pages, and adds the safety affordance for the mixed-mode
override.

Output:
- Verified LiveView for global security profile with override-summary widget.
- Verified per-client edit flow piggybacking on the existing FormComponent.
- New visible warning on ClientsLive.Show when client override is `:none` AND global is
  `:fapi_2_0_security` (the dangerous mixed-mode case).
- Updated `policy_nav` rendered on all four policy pages (PAR, Security Profile, DPoP, DCR).
- LiveViewTest coverage for the global page mount + save + summary, and the show-page warning.
- Human checkpoint at the end to walk through the LiveView UX before signing off.
</objective>
