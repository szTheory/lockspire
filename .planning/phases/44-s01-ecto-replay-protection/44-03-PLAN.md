  <behavior>
    - Client authentication parsing includes validation on JWT lifetimes.
    - If delta between exp and iat/nbf exceeds 10 minutes, the assertion is rejected.
    - If a valid client_assertion has a JTI already recorded, the assertion is rejected.
    - Upon successful authentication with a valid JTI, the JTI is recorded.
  </behavior>
  <action>
    Update `Lockspire.Protocol.ClientAuth` parsing and assertion validation blocks.
    Enforce a strict 10-minute (600 seconds) maximum lifetime (delta between `exp` and `iat` or `nbf`) on any `client_assertion` JWTs.
    Call the `Lockspire.Storage.UsedJtiStore` behavior (using the storage interface) to attempt to `record_used_jti/1` with the `client_id`, `jti`, and `expires_at` (`exp`).
    If recording returns `{:ok, :replay}`, fail the client authentication.
  </action>
