<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| demo seed data -> operator screenshot evidence | Seed data determines what operators and reviewers see during screenshot verification. |
| operator docs -> host adopter | A host adopter may rely on docs to decide what Lockspire owns versus what the host must protect. |
| screenshot inventory -> milestone audit | Screenshot inventory is evidence for visual coverage but must not become runtime product state. |
| static contract tests -> admin LiveView source | Contract tests inspect source and CSS to prevent known UI regressions. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-106-01 | Misrepresentation | demo seeds and screenshots | mitigate | Seed state must cover meaningful admin states and screenshot inventory must name the state exercised per route. |
| T-106-02 | Information Disclosure | screenshots | mitigate | Screenshots use demo-only clients/accounts/tokens and remain evidence artifacts; no secrets or real credentials are introduced. |
| T-106-03 | Spoofing / Elevation of Privilege | operator docs | mitigate | Docs must state the host owns staff authentication and authorization before the admin router. |
| T-106-04 | Regression | admin design system | mitigate | Contract tests reject generic buttons, inline styles, missing CSS primitives, and route/journey drift. |
</threat_model>
