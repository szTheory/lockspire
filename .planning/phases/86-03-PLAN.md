<objective>
Close the proof gap for logout metadata management with repo-native automated coverage for positive and negative RFC 7592 lifecycle cases across the protocol and HTTP controller seams.
</objective>

<scope>
This plan is proof-focused. It expands automated coverage around the shipped management behavior rather than introducing new feature surface.
</scope>

<threat_model>
## Threats

- `T-86-06` The feature may ship with only happy-path coverage, leaving malformed update cases or clear semantics unprotected.
- `T-86-07` Controller and protocol seams could drift, producing different behavior for the same logout metadata updates.

## Mitigations

- Add explicit negative management cases for malformed URI, strict boolean failures, missing paired URI, and front-channel origin mismatch.
- Reuse the same fixtures across protocol and controller tests so both seams prove the same contract.
- Keep the verification command focused on the exact suites that cover the phase requirements.
</threat_model>
