## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| confidential caller -> `/introspect` | Caller credentials and the token string are untrusted until client auth and token lookup succeed |
| introspection protocol -> token persistence | Active response truth must come from stored token state, not inferred policy |
| device client -> `/lockspire/token` and `/lockspire/introspect` | The CLI/device proof crosses the same real HTTP seams that the shipped slice exposes |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-36-04 | Information Disclosure | `Lockspire.Protocol.Introspection` | mitigate | Emit `cnf` only for active tokens and preserve inactive collapse and confidential-caller gating per D-05 and D-06 |
| T-36-05 | Tampering | introspection truth source | mitigate | Read `cnf` only from persisted token records instead of client policy or request-local assumptions per D-07 |
| T-36-06 | Repudiation | device/CLI proof posture | mitigate | Extend the existing generated-host integration seam per D-03 so the public proof chain for CLI-oriented DPoP stays directly executable |
</threat_model>
