Establish the Wave-0 foundations Phase 26 depends on: (1) promote `Lockspire.Clients.generate_client_id/0` from `defp` to `def` with `@spec` so `Registration.register/1` can call it; (2) tighten `Lockspire.Admin.Clients.actor_from_attrs/1` (specifically `normalize_actor_type/1`) so `nil`, blank, and non-atom/non-string actor types raise `ArgumentError` instead of silently defaulting to `:operator` (D-22); (3) audit and update every existing `Admin.Clients` caller to pass an explicit `actor.type`; (4) write the six Wave-0 stub test files mandated by `26-VALIDATION.md` so subsequent waves can attach assertions to existing files; (5) add `Lockspire.Admin.Clients.create_dcr_client/1` — a DCR-aware persistence helper that takes a fully-formed `%Domain.Client{}` (with DCR fields populated upstream by the protocol layer) and writes the row verbatim through `Repository.register_client/1` + `transact_with_audit/2` (the existing `Lockspire.Clients.register_client/1` → `normalize/1` → constructs a fresh `%Client{}` from operator attrs, silently discarding `provenance`, `registration_access_token_hash`, `initial_access_token_id`, `client_id_issued_at`, `client_secret_expires_at`, and the caller-supplied `client_secret_hash` — every DCR must-have would be unreachable through that path); (6) extend `Lockspire.Redaction.for_telemetry/1` and `for_audit/1` drop lists with `:registration_access_token`, `:initial_access_token`, `:rat`, `:iat` (atom + string variants) — belt-and-braces defense via filter to complement D-27's single-sweep defense via test.

Purpose: Without these foundations, every later plan in Phase 26 fails at compile or runtime — `Registration.register/1` can't mint client IDs (Task 1), `actor_from_attrs/1` silently emits `:operator` audit rows for DCR writes (Task 2 — DCR-22 regression), verify commands cite test files that don't exist (Task 3 — Nyquist gate), the persistence path silently strips DCR fields and renders DCR-04/D-17/D-18 must-haves unreachable (Task 4 — blocker resolution), and a future caller stuffing plaintext under `:rat`/`:registration_access_token` slips past today's drop list (Task 5 — defense-in-depth).
Output: A green `mix qa` run with the tightened helpers, the new `create_dcr_client/1` persistence path, the extended Redaction drop lists, and six committed stub test files that downstream waves attach real assertions to.
