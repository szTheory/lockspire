<objective>
Author `Lockspire.Protocol.RegistrationManagement` — the RFC 7592 client configuration management orchestrator. Three public entries: `read/2`, `update/2`, `delete/2`. Each accepts `(client_id_from_url, %Domain.Client{})` where the client is the row matched by `Repository.get_client_by_registration_access_token_hash/1` (a NEW repository helper this plan also delivers). The URL `client_id` and `client.client_id` are compared inside the function — mismatches return `{:error, :invalid_token}` (D-19 enumeration defense). `update/2` reuses the same validator pipeline as `Registration.register/1` (D-02, D-20), rotates the RAT on success, and overwrites the prior RAT hash in the same transaction (implicit invalidation, no deny-list). `delete/2` calls the PUBLIC `Admin.Clients.disable_client/2` with `disabled_by: "dcr_self_delete"`.

Purpose: DCR-11 closes for the management surface here. Phase 27's HTTP controllers will call these three functions; they are designed to be `Plug.Conn`-free so the controllers own only the conn-marshalling and JSON-encoding concerns. The `:self_registered_client` audit actor type is established here; the cross-cutting regression test at plan 26-07 will sweep both `:dcr` and `:self_registered_client` paths.
Output: A 150-200-line orchestrator module + a comprehensive test suite covering happy + sad paths, RAT rotation correctness, the URL/RAT mismatch enumeration defense, audit-row attribution, and telemetry-event assertions.
</objective>

    Step 3 — `update/4` (the most complex):
    ```elixir
    @spec update(String.t(), map(), Client.t(), ServerPolicy.t()) ::
            {:ok, UpdateSuccess.t()} | {:error, Registration.Error.t()} | {:error, :invalid_token}
    def update(client_id_from_url, metadata, %Client{} = client, %ServerPolicy{} = server_policy)
        when is_binary(client_id_from_url) and is_map(metadata) do
      if client_id_from_url != client.client_id do
        emit_unauthorized(client_id_from_url, client)
        {:error, :invalid_token}
      else
        with {:ok, resolved} <- DcrPolicy.resolve(server_policy, nil, metadata),
             :ok <- Registration.validate_intake_metadata(metadata, resolved),
             {new_rat_plaintext, new_rat_hash} <- RegistrationAccessToken.generate(),
             {:ok, updated_client} <- persist_update(client, metadata, new_rat_hash) do
          emit_updated(updated_client)
          emit_rat_rotated(updated_client)

          {:ok,
           %UpdateSuccess{
             client: updated_client,
             registration_access_token_plaintext: new_rat_plaintext
           }}
        else
          {:error, :invalid_client_metadata, info} ->
            error = %Registration.Error{code: :invalid_client_metadata, field: info.field, reason: info.reason, allowed: info[:allowed]}
            emit_update_rejected(client, error)
            {:error, error}

          {:error, %Registration.Error{} = error} ->
            emit_update_rejected(client, error)
            {:error, error}

          {:error, reason} ->
            error = %Registration.Error{code: :persistence_error, reason: reason}
            emit_update_rejected(client, error)
            {:error, error}
        end
      end
    end
    ```
