<objective>
Implement atomic IAT redemption end-to-end:
1. NEW `Lockspire.Storage.Ecto.Repository.redeem_initial_access_token/2` — Postgres `lock("FOR UPDATE")` inside `transact/1`, checks the four freshness axes in D-10 order, marks `used_at = now()` on success.
2. NEW `Lockspire.Protocol.InitialAccessToken.redeem/1` — `Plug.Conn`-free public entry that hashes the plaintext IAT, delegates to the repository, collapses every rejection axis to `{:error, :invalid_token}` per D-11 (discriminator preserved only in telemetry as a `failure_reason` measurement on the `:iat_redemption_failed` event).
3. Extend `test/support/fixtures/initial_access_token_fixtures.ex` with a `persist/1` helper that actually inserts a row (the existing fixture only builds a struct).
4. Write the full test contract for `InitialAccessToken.redeem/1` including a `Task.async` concurrent-redemption test that proves exactly-one-winner.

Purpose: DCR-11 atomicity is the most security-critical primitive in Phase 26 — a non-atomic redemption permits the same IAT to register multiple clients (token reuse). The `lock("FOR UPDATE")` + freshness-check-inside-transaction pattern is already used at `repository.ex:534-557` for `mark_authorization_code_redeemed/2`; this plan mirrors it 1:1 with a 4-axis ladder substituted for the 2-axis ladder. The axis-collapse to `:invalid_token` (D-11) is the IAT-existence-enumeration defense.
Output: Atomic IAT redemption proven by a concurrent-redemption test, with telemetry emitting axis-discriminated failure metadata that contains no plaintext.
</objective>

    ```elixir
    defmodule Lockspire.Protocol.InitialAccessToken do
      @moduledoc """
      Initial access token (IAT) lifecycle — atomic redemption.

      Public entry: `redeem/1` accepts a plaintext IAT, hashes via
      `Lockspire.Security.Policy.hash_token/1`, delegates to
      `Lockspire.Storage.Ecto.Repository.redeem_initial_access_token/2`, and collapses
      every rejection axis (`:not_found | :revoked | :expired | :already_used`) to
      `{:error, :invalid_token}` per Phase 26 D-11. The discriminator is preserved only
      in telemetry on the `:iat_redemption_failed` event as a `failure_reason`
      measurement — never returned to callers, defending against IAT-existence
      enumeration.

      This module is distinct from `Lockspire.Domain.InitialAccessToken` (the defstruct).
      """

      alias Lockspire.Domain.InitialAccessToken, as: Domain
      alias Lockspire.Observability
      alias Lockspire.Security.Policy
      alias Lockspire.Storage.Ecto.Repository

      @spec redeem(String.t()) :: {:ok, Domain.t()} | {:error, :invalid_token}
      def redeem(plaintext) when is_binary(plaintext) do
        hash = Policy.hash_token(plaintext)

        case Repository.redeem_initial_access_token(hash, DateTime.utc_now()) do
          {:ok, %Domain{} = iat} ->
            Observability.emit(:iat_redeemed, %{count: 1}, %{iat_id: iat.id})
            {:ok, iat}

          {:error, reason} when reason in [:not_found, :revoked, :expired, :already_used] ->
            Observability.emit(:iat_redemption_failed, %{count: 1, failure_reason: reason}, %{})
            {:error, :invalid_token}

          {:error, other} ->
            Observability.emit(
              :iat_redemption_failed,
              %{count: 1, failure_reason: :unexpected},
              %{detail: inspect(other)}
            )
            {:error, :invalid_token}
        end
      end
    end
    ```
    CRITICAL: `metadata` passed to `Observability.emit/3` MUST NOT contain `:plaintext`, `:token`, `:iat`, `:initial_access_token`, or any field carrying the plaintext. Only `:iat_id` (the redeemed row's UUID) is safe. The `Lockspire.Redaction.for_telemetry/1` sieve drops `:client_secret`, `:token`, `:token_hash`, `:authorization` but does NOT drop `:initial_access_token` — defense-in-depth means the caller (this module) simply must not pass plaintext fields. The Wave 3 cross-cutting redaction sweep test (plan 26-07) is the closing assertion.
