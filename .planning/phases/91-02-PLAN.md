<threat_model>
## Threats

- `T-91-04` Remote-JWKS diagnosis could remain trapped in telemetry or internal reason codes, leaving operators no actionable way to separate transient upstream issues from unsupported rollover posture.
- `T-91-05` New diagnostics could expose sensitive data such as raw JWT assertions, fetched JWKS bodies, or other secret-adjacent material.
- `T-91-06` Docs or admin wording could overstate Lockspire's support as generic remote-key management instead of the current bounded refresh slice.

## Mitigations

- Build read-only diagnosis output from the shared classified posture produced in Plan 91-01 and present remediation text that maps cleanly to operator actions.
- Keep diagnosis payloads redacted: no raw JWKS JSON, no assertions, no client-secret-derived material.
- Update the host guide, operator guide, and canonical support page together so they all describe the same bounded rotation contract and explicit non-goals.
</threat_model>
