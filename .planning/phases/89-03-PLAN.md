<threat_model>
## Threats

- `T-89-07` Admin creation could stay behind DCR truth, forcing operators into a hidden or asymmetric capability gap.
- `T-89-08` Admin or DCR policy copy could imply that `client_secret_jwt` is a stronger-trust or broad algorithm-configurable posture.
- `T-89-09` New UI truth could accidentally expose secret-derived material or create edit paths that violate the current immutable-field posture.

## Mitigations

- Add `client_secret_jwt` as an explicit operator-create option for confidential clients and keep auth method immutable after creation.
- Show `HS256` as read-only explanatory truth wherever `client_secret_jwt` is displayed; do not add a generic signing-alg editor.
- Keep secret output limited to the current one-time issuance and rotation flows; all stored secret derivatives remain redacted and internal-only.
- Reuse helper-driven policy copy so the DCR page and client detail page describe the same narrow direct-client slice.
</threat_model>
