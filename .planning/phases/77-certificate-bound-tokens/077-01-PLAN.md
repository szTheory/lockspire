<objective>
Bind access and refresh tokens to the client certificate (RFC 8705).

Purpose: Enforce certificate-bound tokens by embedding the `x5t#S256` claim into the token's `cnf` map when mTLS client authentication is used, and verifying this thumbprint on token usage at protected endpoints like Userinfo.
Output: Token issuance context includes `x5t#S256`, and protected resources validate it against the extracted MTLS certificate.
</objective>
