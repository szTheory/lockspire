<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP `Accept` header -> controller negotiation | Untrusted caller preference influences response representation |
| protocol success result -> HTTP response body | Successful introspection output is transformed into either JSON or signed JWT |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-73-04 | Tampering | `Accept` negotiation | mitigate | Parse explicit media types and quality weights, fail safe to JSON on malformed or non-explicit JWT preference per D-03 through D-05. |
| T-73-05 | Information disclosure | inactive JWT responses | mitigate | Preserve the narrow nested inactive payload of only `active: false` per D-10 and D-11. |
| T-73-06 | Repudiation | controller error handling | mitigate | Keep all OAuth error responses on the existing JSON path per D-12 so caller-visible failure semantics stay stable and non-ambiguous. |
| T-73-07 | Denial of service | negotiated signer failure | mitigate | Fail the negotiated JWT path closed to a JSON `server_error` with no partial JWT response, preserving deterministic client-visible behavior and preventing malformed signed output. |
</threat_model>
