end

CRITICAL details:

1. **Filename timestamp `20260507000000`** — must be GREATER than Phase 55's `20260506020000` (verified). Do NOT collide.

2. **`on_delete: :nilify_all`** (D-14) — NOT `:delete_all`. Revoking a ConsentGrant must leave token rows intact for revocation/audit (Doorkeeper rationale).

3. **Partial index** uses `where: "status = 'active'"` (Postgres-evaluated SQL string; Ecto migration `:where` opt accepts raw SQL). Index name `:consent_grants_reuse_idx` (D-19).

4. **`{:array, :map}` resolves to JSONB[]** in Postgres — same idiom as Phase 55's `interactions.authorization_details`.

5. **Defaults**: `authorization_details` defaults to `[]` (matches Phase 55 idiom + avoids `nil` checks downstream). `authorization_details_fingerprint` has NO default (`nil` semantically means "no RAR"; matches `Fingerprint.compute([]) == nil`). `consent_grant_id` on tokens has NO default (`nil` means "no RAR-bound grant" — e.g., client-credentials tokens or pre-Phase-56 tokens).

6. **`create_if_not_exists`** — both indexes use this idempotent form (mirror Phase 55).
