# Phase 47-01: Disable preview bumping, update version to 1.0.0, and upgrade documentation to GA posture

## Objective
Transition Lockspire to its 1.0 GA release posture by updating release configuration, manually bumping the version to 1.0.0, and scrubbing preview terminology from documentation.

## Requirements
- STAB-06: Transition package posture from preview to 1.0 GA (version bump, changelog update, release assets).

## Context
Lockspire is preparing for its 1.0 GA release. It currently uses `release-please` with `"bump-minor-pre-major": true`. To transition to 1.0, we must disable this setting and set the version to 1.0.0. All documentation and the CHANGELOG need to be updated to reflect the GA posture.

## Tasks
- [ ] 1. Update `release-please-config.json`: Set `"bump-minor-pre-major": false`.
- [ ] 2. Update version in `mix.exs` to `"1.0.0"`.
- [ ] 3. Update version in `.release-please-manifest.json` to `"1.0.0"`.
- [ ] 4. Update `CHANGELOG.md` to reflect the transition to 1.0.
- [ ] 5. Scrub preview/beta terminology and update references to 1.0 GA in `docs/install-and-onboard.md`, `docs/maintainer-release.md`, `docs/supported-surface.md`, `README.md`, and `SECURITY.md`.
- [ ] 6. Ensure `test/lockspire/release_readiness_contract_test.exs` passes (or is updated if it contains hardcoded version constraints).

## Success Criteria
- [ ] `release-please-config.json` correctly reflects GA bumping logic.
- [ ] Both `mix.exs` and `.release-please-manifest.json` specify version 1.0.0.
- [ ] All public-facing documentation references 1.0 GA rather than preview.
- [ ] Tests verify the release readiness configuration.
