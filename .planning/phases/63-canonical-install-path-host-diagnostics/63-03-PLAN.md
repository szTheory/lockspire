<objective>
Introduce a manifest-aware upgrade path for Lockspire-managed scaffolding without weakening host-owned file safety.

Purpose: Phase 63 must make regeneration and upgrade predictable. The current generator's blanket no-overwrite rule is safe but too blunt for managed scaffolding. This plan should add a separate `mix lockspire.upgrade` flow that can preview and apply safe updates only where manifest-tracked managed files remain unchanged.
Output: a manifest model, install-time manifest generation, an upgrade Mix task with dry-run output, and tests covering success and refusal cases.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Lockspire upgrade command -> host-edited source files | Wrong ownership or drift detection can destroy host code |
| install manifest -> upgrade decision | Incomplete manifest truth can make unsafe overwrites look safe |
