<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Host extractor -> `EnforceSenderConstraints` | Certificate material crosses from host-managed transport/extraction into Lockspire enforcement. |
| `EnforceSenderConstraints` -> `RequireToken` | Soft plug failure context becomes the sole input to the final HTTP challenge renderer. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-80-07 | Tampering | `EnforceSenderConstraints` | mitigate | Reject MTLS-bound tokens unless the extracted DER certificate hashes to the exact stored `x5t#S256` thumbprint. |
| T-80-08 | Spoofing | MTLS extractor integration | mitigate | Require explicit extractor configuration or explicit `conn.private[:lockspire_mtls_cert]` handoff; never infer proxy trust from global config. |
| T-80-09 | Repudiation / Information Disclosure | `RequireToken` | mitigate | Emit typed but redacted challenge descriptions and logs that distinguish DPoP vs MTLS failures without exposing raw tokens, proofs, or certificates. |
| T-80-10 | Elevation of Privilege | `RequireToken` | mitigate | Map DPoP downgrade, replay, and proof/key substitution errors to `WWW-Authenticate: DPoP` so callers cannot treat those failures as generic bearer token issues. |
</threat_model>
