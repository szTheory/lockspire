# Phase 78: MTLS Discovery, Documentation & Closure

## Overview
This plan fulfills the final requirements of the v1.20 Mutual TLS milestone by accurately advertising mTLS capabilities in the OpenID Connect discovery document, addressing critical security documentation, and finalizing the milestone.

## Requirements Checklist
- [ ] 1. Introduce `mtls_issuer` configuration.
- [ ] 2. Expose `mtls_endpoint_aliases` in the OIDC Discovery document.
- [ ] 3. Ensure MTLS authentication methods are asserted in Discovery tests.
- [ ] 4. Clean up outdated "mTLS out of scope" claims in documentation.
- [ ] 5. Create `docs/mtls-host-guide.md` with security-critical proxy header stripping warnings.
- [ ] 6. Finalize v1.20 milestone closure.

## Step-by-Step Implementation

### Step 1: Configuration & Discovery Updates
1. **`lib/lockspire/config.ex`**
   - Add a new function `mtls_issuer/0` that retrieves `:mtls_issuer` from the `:lockspire` app config (defaulting to `nil`).
   - Ensure it returns a `String.t()` or `nil`.

2. **`lib/lockspire/protocol/discovery.ex`**
   - Update `openid_configuration/0` to inject `mtls_endpoint_aliases` if `mtls_issuer/0` is present.
   - The alias block should selectively replicate endpoint paths using the `mtls_issuer` base URI instead of the standard `issuer`. Valid MTLS aliased endpoints include:
     - `token_endpoint`
     - `revocation_endpoint`
     - `introspection_endpoint`
     - `device_authorization_endpoint`
     - `pushed_authorization_request_endpoint` (par)
     - `userinfo_endpoint`
     - `backchannel_authentication_endpoint` (if enabled)
   - Note: The authorization and registration endpoints typically do not get MTLS aliases.

3. **`test/lockspire/protocol/discovery_test.exs`**
   - Assert that `:tls_client_auth` and `:self_signed_tls_client_auth` are present in `token_endpoint_auth_methods_supported` (and others).
   - Add a test verifying `mtls_endpoint_aliases` correctly generates when `:mtls_issuer` is set.

### Step 2: Documentation & Cleanup
1. **`SECURITY.md` & `docs/supported-surface.md`**
   - Remove outdated bullet points asserting that mTLS is permanently out of scope.

2. **`docs/mtls-host-guide.md` (New)**
   - Draft a comprehensive integration guide.
   - Detail the explicit configuration of the `mtls_issuer`.
   - Explain how to use `Lockspire.MTLS.Extractor.CowboyDirect` and `Lockspire.MTLS.Extractor.ProxyHeader`.
   - **Critical Security Mandate**: Provide bold warnings and examples (e.g., Nginx) illustrating that if `ProxyHeader` is used, the proxy MUST unconditionally strip the incoming client certificate header from external requests to prevent spoofing.

3. **`docs/ecosystem-overview.md` or `README.md`**
   - Add a link to the new `mtls-host-guide.md`.

### Step 3: Milestone Closure
- Update `.planning/ROADMAP.md` marking Phase 78 plans as complete.
- Verify all codebase tasks execute cleanly via `mix test` and `mix credo --strict`.
