  <behavior>
    - `success_response/1` receives `%Lockspire.Protocol.Registration.Success{}` and returns map with `client_id`, `client_secret`, `registration_access_token`, `client_id_issued_at` (epoch), `client_secret_expires_at` (epoch), `registration_client_uri`, and all fields from `client.metadata`.
    - `read_response/1` receives `%Lockspire.Domain.Client{}` and returns the same base structure but WITHOUT `client_secret` and `registration_access_token`.
    - `update_response/1` receives `%Lockspire.Protocol.RegistrationManagement.UpdateSuccess{}` and returns map with `client_id`, NO `client_secret`, but WITH `registration_access_token`, plus epochs and metadata.
    - `error_response/1` receives `%Lockspire.Protocol.Registration.Error{}` and maps `code` atom to string `error` and constructs `error_description`.
  </behavior>
  <action>
    Create `lib/lockspire/web/registration_json.ex` and its test.
    
    In `RegistrationJSON`:
    1. Implement `def success_response(%Lockspire.Protocol.Registration.Success{client: client} = success)`.
    2. Implement `def read_response(%Lockspire.Domain.Client{} = client)`.
    3. Implement `def update_response(%Lockspire.Protocol.RegistrationManagement.UpdateSuccess{client: client} = success)`.
    4. Implement `def error_response(%Lockspire.Protocol.Registration.Error{} = error)`.
    
    To construct the base payload from a `%Client{}`:
    - Map `client.metadata` fields into the root of the output map.
    - Add `client_id: client.client_id`.
    - Add `client_id_issued_at: DateTime.to_unix(client.inserted_at)`.
    - If `client.secret_expires_at` is set, add `client_secret_expires_at: DateTime.to_unix(client.secret_expires_at)`. If nil, set to `0` (RFC 7591 §3.2.1 says 0 means never expires).
    - Add `registration_client_uri: Lockspire.Config.issuer!() <> "/register/" <> client.client_id`.
    
    For `success_response`: Merge the base payload with `client_secret: success.client_secret_plaintext` and `registration_access_token: success.registration_access_token_plaintext`.
    
    For `update_response`: Merge the base payload with `registration_access_token: success.registration_access_token_plaintext`.
    
    For `error_response`: Return `%{error: to_string(error.code)}` and conditionally add `error_description` based on `error.field`, `error.reason`, or `error.allowed`.

    In `test/lockspire/web/registration_json_test.exs`:
    Write tests to explicitly assert the UNIX epoch transformation, the presence of secrets in `success`, the presence of RAT in `update`, and the ABSENCE of secrets in `read`.
