  <behavior>
    - Responds with 404 to all requests when `Lockspire.Config.server_policy().registration_policy == :disabled`.
    - `create/2`: Parses IAT from Bearer token, calls `Registration.register/1`. Returns 201 JSON on success.
    - `show/2`: Parses RAT from Bearer, hashes RAT, queries Repository for Client, validates URL `client_id` matches, calls `RegistrationManagement.read/2`. Returns 200 JSON.
    - `update/2`: Similar RAT lookup, calls `RegistrationManagement.update/2` with body. Returns 200 JSON.
    - `delete/2`: Similar RAT lookup, calls `RegistrationManagement.delete/2`. Returns 204 No Content.
    - Failed RAT lookups and `{:error, :invalid_token}` yield 401 Unauthorized with `WWW-Authenticate: Bearer realm="Lockspire Dynamic Client Registration", error="invalid_token"`.
  </behavior>
  <action>
    Create `lib/lockspire/web/controllers/registration_controller.ex`:
    - `use Phoenix.Controller, formats: [:json]`
    - Add a plug `plug :ensure_dcr_enabled` before all actions. It should check `Lockspire.Config.server_policy().registration_policy` and halt with `send_resp(conn, 404, "")` if it equals `:disabled`.
    
    Helper `extract_bearer_token(conn)`: parse the `authorization` header for `"Bearer " <> token`.
    
    Helper `lookup_client_by_rat(conn)`: extracts token, hashes it via `Lockspire.Protocol.RegistrationAccessToken.hash/1`, and calls `Lockspire.Storage.Ecto.Repository.get_client_by_registration_access_token_hash(hash)`. Returns `{:ok, client}` or `{:error, :invalid_token}`.
    
    Implement `create(conn, params)`:
    - Build `source: %{ip: to_string(:inet.ntoa(conn.remote_ip)), user_agent: List.first(get_req_header(conn, "user-agent"))}`
    - Build `iat: extract_bearer_token(conn)`
    - Call `Lockspire.Protocol.Registration.register(%{metadata: params, server_policy: Lockspire.Config.server_policy(), source: source, iat: iat})`.
    - On `{:ok, success}`, return `conn |> put_status(:created) |> json(Lockspire.Web.RegistrationJSON.success_response(success))`.
    - On `{:error, error}`, delegate to an error mapper (e.g. `handle_error(conn, error)`).
    
    Implement `show(conn, %{"client_id" => client_id})`:
    - Call `lookup_client_by_rat(conn)`. On error, `handle_error(conn, {:error, :invalid_token})`.
    - On `{:ok, client}`, call `Lockspire.Protocol.RegistrationManagement.read(client_id, client)`.
    - Map `{:ok, client}` to `json(RegistrationJSON.read_response(client))`.

    Implement `update(conn, %{"client_id" => client_id} = params)`:
    - Lookup client by RAT.
    - Call `RegistrationManagement.update(client_id, %{metadata: Map.delete(params, "client_id"), server_policy: Lockspire.Config.server_policy(), client: client})`.
    - Map `{:ok, success}` to `json(RegistrationJSON.update_response(success))`.

    Implement `delete(conn, %{"client_id" => client_id})`:
    - Lookup client by RAT.
    - Call `RegistrationManagement.delete(client_id, client)`.
    - On `:ok`, return `send_resp(conn, 204, "")`.
    
    Error handling:
    - Add `defp handle_error(conn, %Lockspire.Protocol.Registration.Error{code: :invalid_token})` returning 401 + `WWW-Authenticate: Bearer realm="Lockspire Dynamic Client Registration", error="invalid_token"`.
    - Add `defp handle_error(conn, {:error, :invalid_token})` doing the same.
    - Add `defp handle_error(conn, %Lockspire.Protocol.Registration.Error{code: :invalid_client_metadata} = e)` returning 400.
    
    Create comprehensive tests in `test/lockspire/web/controllers/registration_controller_test.exs`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Web -> Domain | Token extraction and hashing happens directly at the controller layer before protocol invocation to prevent plaintext tokens from entering memory traces deeper in the stack. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-27-02-1 | Authentication | `RegistrationController` | mitigate | Explicit hashing of `Bearer` token inside `lookup_client_by_rat/1` using identical algorithms as the issuance pipeline. |
| T-27-02-2 | Information Disclosure | `RegistrationController` | mitigate | `invalid_token` and client mismatch uniformly yield 401 Unauthorized to prevent ID enumeration. |
| T-27-02-3 | Elevation of Privilege | `RegistrationController` | mitigate | Checks `registration_policy == :disabled` early and returns 404 to completely drop the request surface area when DCR is turned off. |
</threat_model>
